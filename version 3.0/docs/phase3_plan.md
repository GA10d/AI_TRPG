# Phase 3 开发板

## 需求冻结版

### 1. 游戏主界面
1. 主界面按“核心游玩操作优先”设计，不再是信息堆砌页。
2. 一级可见内容固定为：`最新一轮 AI 叙事`、`玩家输入框`、`最近上下文/历史对话`、`保存`、`读档`、`NPC`、`详情`。
3. `保存 / 读档` 保留在主界面一级入口，不收进详情。
4. `NPC` 作为一级入口，打开独立 NPC 界面，用于立绘展示和人物信息查看。
5. `详情` 承接非核心信息：调试信息、结构化结果原文、模型信息、参数、回放日志、分支图等。
6. 页面视觉以“叙事阅读 + 输入行动”为中心，最新 AI 输出是视觉主焦点，历史对话次级展示，操作区稳定停留在用户手边。
7. 主界面目标是“能长期游玩”的产品页，而不是技术演示页。

### 2. Agent 方案
1. MVP 先做两个 Agent。
2. `Narrator Agent`：负责玩家可见叙事。
3. `Ending Judge Agent`：只判断这一轮后是否进入结局。
4. `Ending Judge Agent` 每轮只吃 `Narrator Agent` 本轮输出，不吃完整上下文。
5. 第一版结构化输出先压到最小，核心只有 `是否进入结局`，必要时再附 `endingState`。
6. `Ending Judge Agent` 的 prompt 放在 `version 3.0/apps/prompt/single_agent/ending_judge_prompt.txt`。

### 3. Narrator Prompt 与开局调用
1. `Narrator Agent` 的 system prompt 以 `version 3.0/apps/prompt/single_agent/narrator_prompt.txt` 为准。
2. 开局时不再额外拼一个新的 `player_info.txt`。
3. 这里的 `player info`，指用户在 `beginning / 角色创建阶段` 写下的文本本身。
4. 正式开始游戏时，首轮输入材料为：`rule.txt`、`story.txt`、`用户填写的 player info 文本`。
5. `system prompt` 与首轮输入材料分离：前者走 `narrator_prompt.txt`，后者走三份输入内容。

### 4. 文生图 Pipeline
1. `3.0` 补一套文生图接口底座，参考 `2.0` 的实现方式先搭通。
2. Prompt 生成方式采用：`业务 prompt + 通用模板 prompt` 拼接。
3. 通用模板按场景类型拆分，至少预留：
- `character_portrait`
- `npc_intro`
- `scene_shift`
- `manual`
4. 这些通用模板需要暴露出来供配置，而不是写死在代码里。
5. 第一阶段先把接口、配置结构、调用链和占位 UI 搭好，后续再接 NPC 立绘和场景图。

### 5. 设置页
1. 设置页除了文本模型配置，还要新增 image provider 配置。
2. 配置范围至少包括：`provider`、`api key`、`base url`、`model`、通用模板 prompt。
3. 这套配置与文本模型配置并列，不混在同一字段里。

### 6. 本轮不主打内容
1. 结构化 Agent 这轮不做复杂状态同步，只做结局判断。
2. 主界面不再默认摊开所有参数、日志和调试数据。
3. 文生图这轮先搭底座，不追求一次做完所有图像玩法。

### 7. 交付附加项
1. 每轮开发完成后，将更新日志补进公告区。

## 2026-04-12 已完成：前端 UI 多语言收口

状态：已完成

### 本轮完成
1. 建立统一的前端 UI 语言包结构，当前提供：
- `zh_cn`
- `en_us`
- `ja_jp`

2. 主菜单右上角新增 UI 语言下拉菜单，玩家可直接切换界面语言。

3. 未实现的语言代码统一回退到英语。
- 例如未来若接入 `hi_in`、`fr_fr`，在对应语言包未完成前，界面默认显示 `en_us`。

4. 主要界面文案已从组件硬编码迁移到语言包读取，覆盖：
- 主菜单
- 继续游戏
- 记录
- 退出
- 游戏主界面
- 剧本选择
- 游戏设置
- 设置页
- 分支回溯图
- App 内建局、占位提示、快速测试等状态文案

5. 修复了日语语言包损坏问题，重建了 `ja_jp.ts`，保证切换到日语后设置页与主流程页面能正确联动切换。

6. 修复了部分旧乱码与本地化相关脏文本：
- 语言选择项显示名
- 建局占位提示
- 快速结局测试文案
- 文本渐显分段正则中的乱码标点

### 当前边界
1. 本轮收口的是“前端静态 UI 文案”。
2. 后端接口直接返回的错误文本、以及模型实时生成的剧情内容，不属于 UI 语言包自动翻译范围。

### 后续建议
1. 后续若继续扩语言，直接在 `apps/web/src/locales/` 下新增对应语言文件即可。
2. 如果希望后端错误也跟随 UI 语言，需要再补一层“服务端错误码 -> 前端语言包”的映射。
