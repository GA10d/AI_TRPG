# AI TRPG 3.0 前端视觉重构设计文档

## 1. 文档目的

本设计文档用于指导 `version 3.0` 前端从“功能可用”升级为“具有明确审美与氛围识别度”的产品界面。

本次视觉重构的目标不是单纯换色，而是建立一套统一、可延展、适合悬疑/惊悚/都市异闻题材的 UI/UX 语言。

本轮主视觉方向：

- 主色系：红 / 黑 / 白
- 氛围关键词：异常信号、档案终端、都市怪谈、夜间广播、机密卷宗、危险提示
- 设计气质：高对比、克制、锋利、时髦、沉浸
- 交互原则：少而狠的动效，清晰的层级，强烈的焦点引导

目标不是做成“普通后台”或“普通游戏启动器”，而是做成一个带有品牌气质的叙事界面系统。

---

## 2. 视觉北极星

### 2.1 品牌感受

用户打开产品后，应该立刻感受到：

- 这不是工具站，而是一个“进入故事异常区”的入口
- 这个产品有自己的世界观气味
- 界面是冷静的，但不是冰冷的
- UI 本身像情报终端、案卷系统、异常广播界面

### 2.2 视觉比喻

建议整套产品统一采用以下视觉隐喻：

- 背景：异常信号场 / 波纹干扰场
- 页面容器：悬浮档案板 / 夜间终端卡
- 按钮：信号触发器 / 指令开关
- 列表：档案抽屉 / 卷宗条目
- 进度条：校准条 / 锁定进度
- 当前选中状态：被标记、被锁定、被点亮

### 2.3 风格原则

- 不做大面积五颜六色的功能色区分
- 不做柔和米色游戏面板风格
- 不做常规 SaaS 后台感
- 不做平均、规整、无主次的栅格布局
- 不做过量特效和持续打扰阅读的动画

---

## 3. 设计系统总纲

## 3.1 色彩系统

建议将整个项目的色彩收敛到 3 个主语义层：

### 颜色角色

- `Black`：承载、背景、结构
- `White`：信息、文字、边线、对比
- `Red`：焦点、CTA、危险、当前状态、系统强调

### 建议色板

```css
--bg-base: #090406;
--bg-deep: #000000;
--bg-panel: rgba(8, 8, 8, 0.74);
--bg-panel-strong: rgba(0, 0, 0, 0.9);
--surface-soft: rgba(255, 255, 255, 0.04);
--surface-mid: rgba(255, 255, 255, 0.08);
--line-soft: rgba(255, 255, 255, 0.12);
--line-strong: rgba(255, 34, 88, 0.34);
--text-primary: #fff8fb;
--text-secondary: rgba(255, 255, 255, 0.72);
--text-dim: rgba(255, 255, 255, 0.52);
--accent-red: #ff255e;
--accent-red-strong: #bf0037;
--accent-red-soft: rgba(255, 37, 94, 0.16);
--accent-red-glow: rgba(255, 37, 94, 0.2);
--signal-dark: #160000;
```

### 色彩规则

- 页面基础背景统一维持暗底，不再出现米色、卡其、棕白大面积底色
- 白色用于主要信息，不要发灰到失去层级
- 红色只用于重点，不要让所有可交互元素都红
- 完成态不要回到绿色，使用白描边 + 红弱化即可
- 错误态也仍然在红谱系内，不额外引入橙黄体系

---

## 3.2 字体系统

本项目包含中文、英文、技术元信息三种文本语境，建议建立三层字体：

### A. 标题字体

用于：

- 页面主标题
- 故事标题
- 启动画面主文案
- 大卡片头部

建议方向：

- 中文：`Noto Serif SC` / `Source Han Serif SC`
- 英文强调：`Space Grotesk` / `Bebas Neue` 仅用于短英文标签

标题应有更强的戏剧感，避免所有标题都用普通系统无衬线。

### B. 正文字体

用于：

- 剧情说明
- 角色概念
- 说明文案
- 设置页描述

建议方向：

- `Noto Sans SC`
- `PingFang SC`
- `Microsoft YaHei`

正文追求稳定可读，不要过于花哨。

### C. 数据/系统字体

用于：

- 进度百分比
- 状态标签
- 元信息
- 系统提示
- 调试信息

建议方向：

- `IBM Plex Mono`
- `Cascadia Code`
- `JetBrains Mono`

这样可以把“故事阅读”和“系统状态”视觉上明显分开。

---

## 3.3 排版层级

### 标题层级建议

- Hero H1：40px - 64px
- Page H1：32px - 44px
- Section H2：22px - 28px
- Card Title：16px - 20px
- Meta Label：12px - 13px uppercase / tracking

### 正文建议

- 主正文：16px - 18px
- 副正文：14px - 15px
- 元信息：12px - 13px

### 排版原则

- 主标题尽量短、强、能立住情绪
- 一屏内不要出现 3 种以上相近字号
- 元信息统一 uppercase 或弱提示格式，形成系统感
- 正文行宽控制在 48-68 个字符的阅读区间

---

## 3.4 形状与边框

### 形状语言

- 主卡片：圆角 20-28px
- 次卡片：圆角 16-18px
- 输入框：圆角 14px
- Badge：胶囊形 999px

### 边框策略

- 默认边框：白色低透明
- 当前选中：红色较强边框
- Hover：边框增强 + 微弱红色辉光
- 危险操作：红底或红描边，不用黄色

整个项目要强调“黑色面 + 细亮边”的高级感，而不是厚重描边。

---

## 3.5 动效系统

### 动效定位

动效不负责炫技，动效负责：

- 建立氛围
- 建立状态切换反馈
- 引导视线

### 动效类型

- 页面进入：淡入 + 轻微上浮，220-380ms
- 面板出现：opacity + blur + y 位移
- 卡片 hover：上浮 1-2px + 边线增强
- CTA：微亮、非大幅弹跳
- 进度条：定向流动 shimmer
- 选中态切换：滑动、擦除、点亮

### 动效原则

- 不给所有元素上 hover
- 不让持续动画和正文竞争注意力
- 游戏阅读页的动画要明显少于菜单和启动页
- 保留 `prefers-reduced-motion`

## 4. 页面级设计方案

## 4.1 Main Menu

对应文件：

- `apps/web/src/components/MainMenuScreen.tsx`

### 当前角色

- 产品入口页
- 新游戏、继续、记录、设置、退出入口
- 公告流展示

### 设计目标

把它从“启动菜单”升级为“异常叙事入口”。

### 推荐布局

```text
左侧：品牌舞台
- 超大标题
- 一句世界观副文案
- 主操作按钮组
- 最近进度摘要

中间：细窄信号分割线

右侧：公告/情报流
- 更新公告
- 版本状态
- 本地默认配置摘要
```

### 视觉建议

- 左侧主标题做得更夸张，允许多行堆叠
- 标题上方放一行小号英文系统标签，例如 `ANOMALY NARRATIVE SYSTEM`
- 主按钮改成大块高对比结构，不要像普通菜单列表
- 右侧公告区更像情报流，可加入日期、分类、优先级
- 分割线可使用竖向发光线，像信号切割层

### 交互建议

- 进入主菜单时，左侧按钮分批出现
- 最近进度可以做成一张“最近进入故事”的悬浮卡
- 公告流支持 hover 高亮，不要只是一串白字卡片

### 应避免

- 所有按钮等宽、等高、等权重
- 左右两栏信息密度完全一样
- 标题与正文视觉张力过弱

---

## 4.2 Story Select

对应文件：

- `apps/web/src/components/StorySelectScreen.tsx`

### 当前角色

- 选择规则
- 选择故事
- 查看故事详情与封面

### 设计目标

把它做成“案件卷宗挑选室”，而不是普通三栏选择器。

### 推荐布局

```text
左栏：规则档案
中栏：故事档案
右栏：大视觉预览 + 关键信息卡
```

### 推荐视觉策略

- 左栏更窄，像分类索引
- 中栏更像真正的“候选故事主列表”
- 右栏必须承担主视觉冲击，应让封面更大、更像展览海报
- 当前选中的故事卡要明显“亮起来”
- 标签、人数、节奏、时长等元信息可重新设计成证物贴纸/机密标签

### 推荐细节

- 规则列表卡：更简短、更硬朗，像档案目录
- 故事列表卡：主标题更大，副信息更收敛
- 右侧详情区：封面、引言、内容警告、支持模式、开场按钮形成强节奏
- Hover 预览可像“预告卡片”，延迟出现没有问题，但视觉要更高级

### 关键优化点

- “开始冒险”按钮必须是全页最强 CTA
- 封面应支持更明显的 hover / click 放大
- 内容警告不要藏在正文里，应该是独立信息模块

---

## 4.3 Game Setup

对应文件：

- `apps/web/src/components/GameSetupScreen.tsx`

### 当前角色

- 开局前配置
- 角色概念录入
- 模型和规则确认
- 开场预览

### 设计目标

把它做成“档案录入台 + 开场校准台”。

### 页面结构建议

```text
左：本局设定与规则参数
中：故事开场预览 / cover / opening
右：角色档案录入 + AI 辅助
```

### 视觉建议

- 中间预览区应更像一个“故事监视器”
- 右侧角色概念输入区做成“档案卡片”，层级高于普通输入框
- AI 按钮维持红色主强调，但要更像系统触发器
- 左侧设置块不要像后台表单，而像“参数模块”

### 交互建议

- 开场预览在加载时可使用更轻的 signal shimmer
- AI 生成 / AI 补全的状态切换要继续保留戏剧化反馈
- 表单不需要太多边框，但需要更强的块级分区

### 推荐新增视觉元素

- 角色档案编号
- 当前选择故事的小型封面预览
- 本局风险标签
- 模型状态灯式摘要

## 4.4 Bootstrap / Loading

对应文件：

- `apps/web/src/components/GameBootstrapScreen.tsx`

### 当前角色

- 创建会话
- 生成开场
- 等待 narrator 返回

### 设计目标

把它做成一个有沉浸感的“信号锁定界面”，而不是简单等待页。

### 页面语言

- 封面图更大
- 标题更夸张
- 进度条更像校准条
- 提示轮播更像系统广播内容

### 建议模块顺序

1. 故事封面
2. 当前主标题
3. 一句叙事型说明
4. 进度条
5. 当前阶段详细说明
6. 等待提示
7. 系统 tips

### 建议增强项

- 将 `steps` 做成“点亮式阶段轨”
- 当前阶段可加入红色 pulse
- 背景波纹在 bootstrap 时强度最高
- 封面周围增加轻微红色边缘辉光

## 4.5 Game Screen

对应文件：

- `apps/web/src/components/GameScreen.tsx`
- `apps/web/src/components/PlaythroughGraphPanel.tsx`

### 当前角色

- 故事主阅读
- 历史记录
- 输入行动
- 存档 / NPC / 细节抽屉

### 设计目标

把它从“游戏信息页”升级成真正的“阅读舞台 + 指令台”。

### 核心结构建议

```text
顶部：故事标题 / 当前状态 / 核心操作

主区：
左：当前叙事主舞台
右：历史与上下文

底部：固定输入指令台

侧抽屉：
- Save / Load
- NPC
- Details
- Graph
```

### 主阅读区建议

- 当前 narration 必须成为视觉中心
- 正文区更像阅读器，而不是普通卡片
- 正文字体尺寸和行高要明显优于其余区域
- 当前回合文本容器可有更深的背景和更宽的留白

### 历史区建议

- 玩家发言与 narrator 内容必须有结构差异
- 推荐使用：
  - 玩家：白描边 / 白低透明块
  - GM：黑底叠红弱高光
- 历史列表应像“时间切片”，不是平铺日志

### 输入区建议

- 输入框做成真正的底部控制台
- 提交按钮永远维持强 CTA 感
- 加一个次级说明行，例如当前状态、可用性、限制说明
- 如果锁定输入，应明确显示是“系统正在接入”而不是普通 disabled

### 抽屉系统建议

- 抽屉不是普通侧栏，而是“次级工作台”
- 每个抽屉打开时都像切换到子系统
- 标题、meta、工具按钮、内容区有明显分层

### Playthrough Graph 建议

- 节点图应该视觉上更“神经网络 / 红线关系图”
- 当前节点像被锁定的异常点
- 分支连线可以更细、更暗、更像档案关系图

  - 当前轮次
  - 最近更新时间
  - 当前状态
- 主操作按钮更直接，辅助操作弱化

### 视觉关键词

- 续作入口
- 现场恢复
- 最近信号

## 4.8 Settings

对应文件：

- `apps/web/src/components/SettingsScreen.tsx`

### 设计目标

从“后台配置表单”升级成“系统控制台”。

### 结构建议

- General
- Text Model
- Image Model
- Image Prompt Template

每个区域都应成为独立面板，不要像一大张长表单。

### 视觉建议

- 模型项可以做成“设备卡”
- 当前 profile 高亮更明显
- `configured / missing / override` 用系统标签形式表示
- 输入区和只读说明区要拉开视觉层次

### UX 建议

- 默认值、当前值、覆盖值必须一眼区分
- 复杂配置尽量做模块，而不是密集输入框
- 将说明文案收口成“说明卡片”

这是收尾页，不应只是“退出提示”。

### 推荐氛围

### 建议设计

## 5. 组件级设计规范

## 5.1 按钮

### 主按钮

- 红底白字
- Hover 时边缘发光，不大跳
- 主 CTA 可更大、更宽

### 次按钮

- 黑底白描边
- Hover 时边线增强

### 危险按钮

- 深红底
- 不额外引入黄色

---

## 5.2 卡片

卡片分 3 层：

- L1：主卡片，承载主要内容
- L2：信息卡，承载补充内容
- L3：状态卡，承载小型 meta / tag / 说明

### 规则

- L1 背景更深
- L2 背景更轻
- L3 更薄、更像标签容器

---

## 5.3 表单与输入

### 输入框

- 黑底
- 白字
- 红色 focus ring
- placeholder 偏灰白

### 说明文本

- 不要与正文太像
- 维持系统说明的弱层级

### 开关与下拉

- 当前选中状态必须清楚
- 下拉菜单建议使用深色浮层

---

## 5.4 Badge / Status

只保留几种清晰语义：

- `active`：红
- `normal`：白
- `dim`：灰白
- `danger`：更亮的红

避免出现大量彩色 badge。

---

## 5.5 进度条

适用于：

- Bootstrap
- AI 处理中
- 图像生成中

建议风格：

- 深色轨道
- 红到白渐变填充
- 定向 shimmer
- 百分比以 mono 字体显示

---

## 6. 背景系统

## 6.1 统一背景策略

建议背景分 3 档：

### A. 强沉浸背景

用于：

- Main Menu
- Bootstrap

策略：

- 使用动态波纹背景
- 对比强
- 动效允许更明显

### B. 中沉浸背景

用于：

- Story Select
- Game Setup

策略：

- 保留波纹，但降低对比、减慢扰动
- 提高面板不透明度，保证阅读

### C. 弱沉浸背景

用于：

- Game 主阅读
- Settings
- Continue
- Records

策略：

- 背景运动要更轻，甚至可静态化
- 阅读与操作优先，氛围退后

---

## 7. 响应式原则

### Desktop

- 保留舞台感与大留白
- 主次区域明显分级

### Tablet

- 三栏收敛成两栏
- 抽屉替代侧栏

### Mobile

- 减少并列结构
- 让阅读区先行
- 把辅助信息收进折叠区和 modal
- CTA 保持大而清楚

### 移动端注意点

- 波纹背景继续存在，但动态更轻
- 主内容卡片宽度尽量大，避免过多左右留白
- 输入区固定底部时要保证键盘弹出可用

## 8. 无障碍与可用性

即使风格更强，也必须保证：

- 正文和背景有足够对比
- 按钮不只靠颜色区分
- 当前选中态可通过边框/图标/位置共同表达
- `prefers-reduced-motion` 生效
- 键盘操作路径清晰
- Hover 不是唯一信息入口

特别注意：

- 游戏主阅读区不要因为背景太强导致疲劳
- 红色不应用于大段正文
- 长文阅读必须稳定，不要被装饰性效果干扰

## 10. 对当前分支的建议结论

如果这个 branch 的主要目的就是“前端视觉优化”，建议不要再做零散修补，而是直接按以下方向推进：

1. 先锁定“红黑白 + 异常信号场”的大方向
2. 先做 `MainMenu + StorySelect + GameScreen`，这三页最能显著提升质感
3. 让 `Bootstrap` 成为品牌级过场页
4. 把 `Settings / Continue / Records` 从普通功能页升级成同一视觉系统下的模块页

最终目标不是“更好看一点”，而是：

- 用户一眼能记住这个产品
- 截图拿出来就有辨识度
- UI 本身能参与叙事氛围
- 所有界面看起来像同一个世界，而不是若干能跑的页面
