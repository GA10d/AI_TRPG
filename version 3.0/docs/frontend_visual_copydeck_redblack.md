# AI TRPG 3.0 前端视觉文案方案

## 1. 文档定位

本文件是 `docs/frontend_visual_reboot_redblack.md` 的配套文案稿。

用途：

- 为红白黑主视觉方向补齐页面级文案
- 为后续 `zh_cn.ts` 重写提供统一语气和文本母版
- 让视觉改版不只是“换皮”，而是连同品牌语气一起升级

本文件采用“设计师视角 + 产品可落地”的写法。文案目标不是说明功能，而是增强：

- 世界观氛围
- 页面辨识度
- CTA 清晰度
- 微交互的高级感

---

## 2. 文案总原则

## 2.1 语气关键词

- 冷静
- 克制
- 都市异闻
- 异常信号
- 系统广播
- 危险但不油腻

## 2.2 写法原则

- 少说废话，避免“欢迎使用”“您可以在这里”
- 尽量让每句文案都带一点画面感
- 标题短，副标题稳，提示文案狠
- 系统状态文案优先使用“正在接入 / 正在锁定 / 正在回传 / 已记录”这类语义
- 避免过度中二和夸张网文腔

## 2.3 三层文案结构

### A. 舞台文案

用于：

- 页面主标题
- Hero 描述
- Loading 标题

要求：

- 强记忆点
- 情绪明确
- 不直接解释功能

### B. 功能文案

用于：

- 按钮
- Tab
- 表单标签
- 区块标题

要求：

- 简短清晰
- 少修饰
- 一眼能懂

### C. 系统文案

用于：

- 进度提示
- 状态标签
- 空状态
- 提示和错误

要求：

- 语义明确
- 有系统感
- 不啰嗦

## 4.4 Bootstrap / Loading

对应页面：

- `GameBootstrapScreen`

### 主标题建议

根据阶段轮换显示，建议文案比现在更具“信号锁定感”。

#### 阶段标题建议

- 进入页面：`正在进入故事核心区`
- 加载内容：`正在读取卷宗`
- 组装 Prompt：`正在校准叙事入口`
- 请求 Narrator：`正在呼叫叙事端`
- 等待首条回复：`等待首条叙事`
- 最终写入：`正在封存本局现场`

### 副标题 / loading hint 建议

- `所有线索已经在路上。请保持在线。`
- `故事正在成形，最先回来的通常不是答案。`
- `当第一条 narration 返回时，这局才算真正开始。`

### 进度区文案

- 区块标题：`接入进度`
- 百分比格式：`{percent}%`

### detail 文案风格

推荐格式：

- `系统已读取规则文本，正在等待叙事端返回首段内容。`
- `角色设定与故事材料已封装完成，正在准备正式开场。`

### wait 提示建议

- `首次加载通常需要一点时间。越危险的故事，越不会立刻回答。`

### tip 区文案建议

作为轮播内容，可用以下 8 条：

1. `详情面板会在后续开放结局判定、回放日志与分支图。`
2. `NPC 面板后续会接入角色立绘与档案检索。`
3. `当开场正式回传后，你就可以提交第一轮行动。`
4. `本轮加载不会中断本地记录，异常关闭后仍可继续恢复。`
5. `如果长时间没有返回，优先检查模型接入状态。`
6. `本项目的主体验仍然是文本叙事，而不是数值堆叠。`
7. `每一轮行动都会改变后续分支，不一定会改变真相。`
8. `越短的输入不一定越差，但越具体的行动通常越有戏。`

## 5. 组件与微文案

## 5.1 全局按钮文案风格

建议统一成“动作导向”：

- 用 `进入 / 接入 / 写入 / 恢复 / 读取 / 锁定`
- 少用 `确定 / 提交 / 下一步 / 点击这里`

示例：

- `开始新局` 优于 `开始游戏`
- `进入故事` 优于 `确认`
- `恢复现场` 优于 `继续`
- `写入默认设置` 优于 `保存设置`

## 5.2 空状态文案模板

结构建议：

- 第一层：说明现在没有内容
- 第二层：指出下一步或未来价值

模板：

- `这里还没有内容。`
- `当你完成第一局后，这里会开始积累档案。`

## 5.3 错误状态文案模板

建议统一风格：

- 不甩锅
- 不技术化过头
- 明确下一步

模板：

- `信号暂时中断，请稍后重试。`
- `本次回传失败，你可以重新生成或继续手动推进。`
- `当前入口不可用，请先检查模型接入配置。`

## 5.4 Loading 文案模板

避免重复“加载中...”

建议替换为：

- `正在接入...`
- `正在读取...`
- `正在回传...`
- `正在写入...`
- `正在锁定...`

## 6. 推荐落地方式

## 6.1 实施顺序

建议按下面顺序把文案真正写进前端：

1. `MainMenu`
2. `StorySelect`
3. `GameSetup`
4. `Bootstrap`
5. `GameScreen`
6. `Settings`
7. `Continue / Records / Exit`

## 6.2 locale 改写建议

建议不要一次性全量改写 `zh_cn.ts`，而是按页面逐步替换。

优先替换的 key：

- `mainMenu`
- `storySelectScreen`
- `gameSetupScreen`
- `gameBootstrapScreen`
- `gameScreen`

## 6.3 文案审校规则

每次替换后检查：

- 标题是否足够短
- 副标题是否在 1-2 句内完成说明
- CTA 是否一眼知道动作
- 空状态是否不显得敷衍
- 系统状态是否保持统一口吻
