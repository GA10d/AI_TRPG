今天的内容生成器排查与生成链路更新已经整理如下：

1. 内容生成器现在会更清楚地暴露图片生成状态；如果封面或配图走到了 mock / fallback 占位图，会在任务 warning 和运行日志里直接提示，方便判断是图片模型配置、API key、base URL 还是模型名需要检查。
2. 项目运行日志已经扩展为更细的 JSONL 链路日志，覆盖 HTTP 请求、Session 创建与回合提交、内容生成任务、图片生成开始 / 成功 / fallback / 失败等关键节点；默认写入 `local_data/runtime_logs/`。
3. 封面生成 prompt 已加入 16:9 横向宽屏画幅要求，正常 prompt 和兜底 prompt 都会保留这个限制，后续生成封面会更贴近剧本选择页和展示位的比例。
4. 内容生成器图片相关 prompt 已经拆成本地文件，集中放在 `apps/prompt/image_generation/`，包括封面、NPC 肖像、配图和资产规划 prompt，之后可以直接改文本文件微调生成风格。
5. 图片生成仍保留安全兜底：本地 prompt 文件缺失或为空时会回退到内置 prompt，避免内容生成流程因为模板误删而中断。
