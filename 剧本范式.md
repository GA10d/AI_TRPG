# 剧本范式

**必须项**

1. 剧本元信息

   * 剧本名
   * 剧本ID/版本号
   * 题材标签：悬疑 / 恐怖 / 校园 / 奇幻 / 科幻 / 恋爱 / 冒险 / 末日等
   * 风格标签：轻松 / 压抑 / 写实 / 浪漫 / 黑暗 / 高风险等
   * 适用人数：单人 / 双人 / 多人
   * 推荐流程长度：短篇 / 中篇 / 长篇
   * 推荐节奏：快节奏 / 中速 / 慢热
   * 推荐主持风格：强引导 / 弱引导 / 高自由 / 重氛围
   * 内容边界提示：是否含血腥、惊吓、情感压迫、伦理冲突等

**2. 故事Intro（用于显示）**  
一句话或一小段说明：  
玩家是谁  
身处什么局面  
当前最核心的问题是什么  
剧情的主要张力来自哪里  
这是剧本最重要的入口摘要。  
例如：  
“你追踪盗掘者进入即将被沙暴吞没的古城遗迹，必须在有限时间内破解机关并决定遗迹核心的去留。”  
“你在调查一起伪装成自杀的命案时，发现新的知情人正在被同一凶手灭口。”  
“这个初夏，你被不同的人以不同方式喜欢着，必须在关系变化中逐步做出回应。”

但是要比这个详细一点，这部分相当于玩家刚进入游戏会看到的intro。

**3. 玩家角色定位（有就写，没有就不写）**

* 玩家身份
* 初始处境
* 初始认知范围
* 初始目标
* 初始限制
* 初始优势
* 是否固定主角
* 是否允许自定义背景
* 是否有预设关系网
* 兼容不同题材时建议用以下字段：
* 身份标签
* 社会位置
* 与核心事件的关系
* 与关键 NPC 的初始关系
* 玩家知道什么 / 不知道什么

**4. 核心目标**  
把剧本目标结构化。  
建议拆成：  
显性目标：玩家表面上要做什么  
隐性目标：实际深层目标是什么  
阶段目标：中途必须完成哪些推进任务  
成功条件  
失败条件  
不可逆损失条件（如果有）  
这部分很关键，因为 agent 要靠它判断推进方向。

**5. 剧本主推进轴**  
这是剧本层非常重要、但容易漏掉的一项。  
即：这部剧本主要靠什么推进？  
可选类型：  
调查推进  
解谜推进  
生存推进  
情感推进  
战斗推进  
社会博弈推进  
旅行探索推进  
任务清单推进  
多线并行推进  
也可以复合：  
“调查 + 时间压力”  
“情感推进 + 分支选择”  
“探索解谜 + 生存风险”  
这一项会直接影响 agent 怎么组织信息和选项。

**6. 场景 / 节点结构**  
这是剧本的核心骨架。  
每个节点建议至少包含：  
节点名  
节点类型：开场 / 探索 / 对话 / 战斗 / 谜题 / 休整 / 结局 / 转折  
节点功能：提供线索 / 推进关系 / 施加压力 / 制造误导 / 完成收束  
场景描述  
可见元素  
可交互对象  
可获得信息  
可触发事件  
进入条件  
离开条件  
节点风险  
指向下一步的钩子  
这部分建议做成剧本最主要的组织方式。

**7. 关键实体**  
不要按题材写死成“NPC/怪物/嫌疑人/恋爱对象”，统一写成**关键实体**最稳。  
每个关键实体建议包含：  
名称  
类型：NPC / 敌对者 / 同伴 / 关系对象 / 组织 / 怪物 / 场景意识体 / 中立者  
表层印象  
外貌（英文prompt用于文生图）
真实动机  
立场  
与玩家关系  
知情范围  
对玩家隐瞒的信息  
可提供的信息/资源  
会制造的阻碍  
行为触发逻辑  
变化条件  
退场条件  
这样刑侦、恋爱、恐怖、奇幻都能通用。

**8. 关键信息单元**  
每条信息包含：  
信息名  
类型：线索 / 证言 / 物证 / 设定知识 / 情感信号 / 谣言 / 错误信息 / 预兆  
来源  
获取方式  
可信度  
作用  
关联对象  
是否为核心信息  
是否需要交叉验证  
是否会误导  
缺失时是否需要补提示  
这能兼容：  
刑侦本里的证据链  
恐怖本里的异象与民俗知识  
恋爱本里的行为信号与情感线索  
冒险本里的机关提示与地图信息

**9. 触发器与状态变化**  
说明“什么情况下故事会变”。  
建议至少写：  
触发条件  
触发后事件  
影响范围  
持续时间  
是否可逆  
是否会改变 NPC 态度 / 场景状态 / 风险等级 / 结局可达性  
常见触发器：  
玩家抵达某场景  
玩家获得某信息  
玩家与某人对话到某层  
玩家拖延时间  
玩家做出某种选择  
玩家资源见底  
玩家触发错误操作  
玩家明确站队

**10. 风险与后果（剧本内具体化）**  
规则层已经有“风险与后果机制”，但剧本里要写成具体表现。  
建议包含：  
核心风险源  
风险升级方式  
即时后果  
延迟后果  
隐性后果  
长期后果  
对剧情推进的影响  
对结局的影响  
例如：  
被敌对者察觉  
错过关键时间窗口  
关系破裂  
新死者出现  
恐惧加深  
资源耗尽  
结局路线锁死

**11. 分支点**  
很多剧本不是“自由推进”，而是由少数关键分歧点改变结构。  
每个分支点建议写：  
分支点名称  
触发时机  
玩家可做选择  
每个选择的直接后果  
每个选择的延迟后果  
会影响哪些角色/节点/结局  
是否可回头  
是否为隐性分支

**12. 结局结构**  
建议不要只写结局文本，而写成结构。  
包含：  
结局名  
结局类型：胜利 / 和解 / 生还 / 真相达成 / 关系成立 / 失败 / 牺牲 / 开放  
达成条件  
必要前提  
禁止条件  
收束对象  
最终揭示内容  
主题落点  
另外建议补一项：  
未完成支线如何收束  
这样 agent 在收尾时不会漏角色、漏情节。

**13. 主持约束 / Agent 约束**  
这项必须单独列出来，不要混在别处。  
建议包含：  
是否允许强引导  
是否允许主动补关键线索  
是否允许保护性兜底  
是否允许模糊判定  
是否允许压缩场景  
哪些信息不能提前说  
哪些结论不能替玩家做  
是否需要确保某类体验成立  
是否必须保留悬念  
是否必须避免某类走向  
这对 LLM 主持极其重要。

**二、可选项**  
**1. 时间结构**  
适合调查、恐怖、生存、灾难类。  
可包含：  
全局时间轴  
倒计时  
时间节点事件  
错过窗口的后果  
白天/夜晚差异  
特定时刻触发的变化

**2. 状态 / 计量模块**  
如果剧本本身依赖某种状态值，就单独列。  
例如：  
恐惧  
理智  
污染  
曝光度  
关系亲密度  
信任  
嫌疑度  
沙蚀程度  
饥饿 / 寒冷 / 疲劳

**3. 特殊机制模块**  
用于题材特化。  
例如：  
区域专属能力  
民俗禁忌规则  
证据闭环要求  
情感线解锁机制  
幻觉机制  
潜入机制  
追逐机制  
社会舆论机制

**4. 道具 / 物件表**  
适合解谜、冒险、恐怖、刑侦类。  
建议字段：  
名称  
类型  
外观  
来源  
用途  
可组合对象  
是否关键道具  
是否可误用  
是否消耗品

**5. 敌对系统 / 怪物系统**  
如果剧本存在明确敌对压力。  
建议字段：  
敌对者名称  
威胁来源  
行为模式  
弱点  
出现场景  
升级条件  
压迫方式  
退场方式

**6. 关系网络**  
适合恋爱、社交、宫斗、派系、悬疑本。  
建议字段：  
实体之间的关系  
明面关系  
隐藏关系  
冲突点  
利益链  
可变化关系  
玩家介入点

**7. 误导信息 / 红鲱鱼表**  
对悬疑、恐怖、冒险很好用。  
包含：  
误导信息  
看似支持什么判断  
实际真相是什么  
如何被纠正  
如果玩家信了会发生什么

**8. 演出文本**  
建议单独放，不要混进结构层。  
包括：  
开场白  
场景朗读文本  
惊吓文本  
梦境文本  
关键 NPC 台词样例  
结局朗读文本  
这些对主持体验有帮助，但不属于结构主干。

**9. 示例流程**  
如果要降低 agent 使用门槛，可以加。  
例如：  
一个标准调查回合示例  
一个关系推进场景示例  
一个节点切换示例  
一个提示递进示例  
一个结局收束示例

**10. 变体与扩展**  
用于复用和多轮游玩。  
包括：  
替换真相  
可替换角色  
可替换结局  
难度档位  
多周目变化  
单人/多人适配差异

**三、我建议你最后整理成的剧本模板骨架**  
你可以直接按这个顺序组织：  
**剧本必须项**  
剧本元信息  
故事 premise  
玩家角色定位  
核心目标  
主推进轴  
场景/节点结构  
关键实体  
关键信息单元  
触发器与状态变化  
风险与后果  
分支点  
结局结构  
主持/Agent 约束  
**剧本可选项**  
时间结构  
状态/计量模块  
特殊机制模块  
道具/物件表  
敌对系统/怪物系统  
关系网络  
误导信息表  
演出文本  
示例流程  
变体与扩展

**四、一个很关键的建议**  
为了兼容各题材，字段命名尽量抽象化，不要写死：  
不要只写“嫌疑人”，写“关键实体”  
不要只写“线索”，写“关键信息单元”  
不要只写“怪物”，写“敌对系统/威胁实体”  
不要只写“恋爱对象”，写“关系对象”  
不要只写“机关”，写“特殊机制/节点交互”  
这样你的 schema 才能同时装下：  
刑侦本  
恋爱本  
恐怖本  
冒险本  
奇幻任务本

如果你下一步要继续落地，我建议直接做成：
\*\*“剧本字段名 + 字段说明 + 是否必填 + 示例”\*\* 的标准表。
