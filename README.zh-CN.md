## 项目简介

这是一个个人独立项目，完全出于我对角色扮演游戏与 AI 故事生成器的兴趣而持续推进。

我想探索的并不只是“让 AI 讲故事”，而是让 AI 真正支撑一个可以游玩、可以收束、可以沉浸、并且能够不断扩展的 TRPG 体验。

## Version 2.0 做了什么

`version 2.0` 主要围绕单人体验与单 Agent 叙事闭环展开，建立了较稳定的文本跑团原型，并为后续的内容扩展、状态维护与回放分析打下了基础。

它已经能够在 single agent 模式下较稳定地推进剧情并触发结局，使整个游戏体验形成闭环。对于 AI 角色扮演游戏来说，这一点很关键，因为这意味着 AI 不只是不断续写文本，而是真正能够掌握故事节奏，知道什么时候该把故事讲完。

## Version 3.0 做了什么

`version 3.0` 在保留 single agent 的基础上，进一步引入了 multi-agent system (MAS) 框架，让不同模型分别负责 NPC、世界与事件、宏观剧情安排与节奏，以及具体叙述等不同层面的工作。

与此同时，`version 3.0` 也把整体体验继续往完整产品形态推进，包括更现代的前后端结构、更清晰的内容组织方式，以及面向文本与漫画协同生成的工作流基础。

## 我已经完成的内容，以及正在尝试攻克的问题

### 1. 更丰富的多模态用户体验

目前已经实现了漫画与文本的配套生成，也在一定程度上解决了角色一致性问题。

但实际使用中，不同 image model 在同一个框架下的效果差距非常大，很难得到一个在价格、时间与质量之间都足够理想的 trade-off。当前我正在着手改进文生图 pipeline，希望先用更普通、成本更可控的模型，生成更稳定、更可用的图像结果。

在这之后，我也计划继续尝试视频与 AI 音频能力，把体验从“文本 + 漫画”进一步扩展到更完整的多媒体叙事。

### 2. 故事结局与游戏闭环

在 `version 1.0` 和 `version 2.0` 中，single agent 模式都能够较稳定地达成故事结局，因此整体游戏体验是闭环的。过去在人类玩家参与的测试里，故事通常会在 20 轮以内触发结束。

`version 3.0` 除了保留 single agent，还加入了 MAS 框架来讲述故事，并增加了 AI 队友与 AI 玩家选项。AI 队友支持私聊，玩家可以通过私聊影响队友行为；而在 AI 玩家模式下，系统也可以让 AI 自主参与这套叙事框架。

不过目前在 MAS 框架下，AI 玩家即使经历了 40 轮，仍然没有稳定触发故事结局，这说明多 Agent 协同之后，节奏控制与结局收束仍然是一个尚未解决的问题。由于精力有限，我目前还没有对 MAS 框架进行系统的人类玩家测试。

### 3. 长线叙述中的逻辑一致性

AI 在长线故事叙述中的逻辑一致性，一直是故事创作里最困难的问题之一。只有真正解决了长线叙述中的幻觉问题，以及 AI 过于迎合、过于谄媚用户的问题，玩家才能获得更强的沉浸感。

目前，MAS 系统已经在一定程度上解决了“AI 过于讨好玩家”的问题。现在的 NPC 会更明确地从自身利益出发进行交互，甚至会为了自己的利益去欺骗、背叛或利用玩家。

至于逻辑一致性，我曾尝试让一个额外的 LLM 持续维护世界模型：通过结构化输出，从文本中不断摘取事实与关键检查点，再持续更新世界状态，以此减少叙事漂移。但这个方案会显著增加 token 消耗，因此我最终放弃了这条路线。
